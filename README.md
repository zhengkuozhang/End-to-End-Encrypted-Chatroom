# 端到端加密P2P局域网聊天室   End-to-End Encrypted Chatroom

**一款基于 Python 开发的去中心化、纯点对点、端到端全加密局域网聊天工具**，专为隐私安全私密通信量身打造。全程无需部署任何中心化服务器，局域网设备点对点直连通信，支持 AES 高强度消息加密、本地聊天记录安全持久化、点对点无损文件直传，杜绝数据抓包、监听嗅探和云端数据泄露风险，兼顾通信隐私性、使用便捷性和运行稳定性。

---

##  核心特性 Core Features

###  去中心化网络架构 Zero Centralization

彻底摒弃传统客户端-服务器（Client-Server）通信模型，所有运行本程序的设备均为网络平等独立节点（Peer），不存在数据中转、服务托管第三方节点，从架构根源规避数据集中泄露隐患。采用**UDP 广播协议（9999端口）**完成局域网内所有在线节点自动扫描探测与节点魔法握手认证，无需手动配置IP地址即可自动发现同网络在线设备；依托**TCP 长连接协议（8888端口）**搭建稳定可靠的点对点专属数据传输隧道，保障聊天消息、文件数据传输不丢包、不断连。

###  高强度端到端全量加密 End-to-End Encryption

深度集成 cryptography 加密工具库核心 Fernet 加密模块，实现通信数据全链路安全防护。所有对外传输的数据包包含消息头部校验信息（Header）和聊天正文内容（Body），在设备网卡发送前完成**URL安全Base64编码的AES对称加密处理**，局域网内任何第三方均无法通过抓包工具、网络嗅探设备窃取、破解通信内容，彻底杜绝局域网监听、数据窃听、信息篡改等安全风险，全程保障聊天内容私密性。

###  数据绝对私有化本地存储 Local Data Ownership

坚守数据本地权属原则，所有聊天记录、通信交互数据永不上云、永不对外同步。程序内置轻量级 SQLite3 嵌入式数据库，无需额外安装部署数据库服务，所有会话消息、聊天历史均独立持久化存储在本地专属 `chat_history.db` 数据库文件中，实现物理级别的数据隔离，仅设备本机可查看管理聊天数据，从存储层面杜绝隐私数据外泄。

###  无缝点对点文件直传 Seamless File Transfer

支持局域网节点之间不限格式、不限大小的快速文件点对点直接传输，无需借助网盘、传输工具等第三方媒介，文件数据全程跟随加密隧道同步传输，安全不泄露。所有接收的文件会自动按照时间戳分类归档、隔离存储，自动创建本地专属 `P2P_Downloads` 下载目录统一管理接收文件，无需手动配置保存路径，便捷高效且文件安全可追溯。

###  高性能抗阻塞图形化界面 Thread-Safe GUI

基于Python原生Tkinter模块开发跨平台图形化操作界面，无额外复杂UI依赖，Windows、macOS、Linux全系统适配，开箱即用无需繁琐配置。底层采用标准线程安全队列 queue.Queue 核心机制，彻底解耦异步网络I/O通信操作与主线程UI界面渲染工作，有效避免网络传输、消息收发导致的界面卡顿、卡死、闪退问题，保障聊天操作全程流畅稳定，交互体验良好。

## 快速启动指南

### 1. 环境准备

确保本地设备已安装 **Python 3.7 及以上稳定版本**，配置好Python系统环境变量，保证cmd、终端可正常调用python与pip指令。

### 2. 克隆项目仓库

打开电脑终端或Git Bash工具，执行以下Git指令，克隆项目源码至本地并进入项目根目录：

```bash
git clone https://github.com/zhengkuozhang/End-to-End-Encrypted-Chatroom.git
cd End-to-End-Encrypted-Chatroom
```

### 3. 安装项目依赖库

项目运行需加密、网络请求、网页解析相关第三方依赖，执行以下pip指令一键批量安装所需依赖包：

```bash
pip install cryptography requests beautifulsoup4
```

### 4. 启动加密P2P聊天室

将需要通信的多台设备连接**同一局域网（同一Wi-Fi/同一局域网网段）**，所有设备均完成上述环境配置后，在每台设备终端执行以下指令运行核心主程序：

```bash
python P2P_Chat_Ultimate.py
```

### 使用提示

程序启动后会自动通过UDP广播扫描局域网内所有在线节点，无需手动添加IP。当其他设备节点上线后，程序系统消息区域会自动弹出上线提醒通知，节点连接成功后即可开启**端到端加密安全聊天、点对点无损文件传输**所有核心操作。
